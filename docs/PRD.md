> **文档版本：** 1.0
> **最后更新：** 2026-04-22
> **Owner：** 张子超

---

## 1. 产品概念

### 1.1 一句话描述

**TNT 是一个 macOS 全局语音输入法——按下快捷键说话，松开即完成输入，无需切换输入法。**

灵感源自老罗的 TNT 工作站，但 TNT 专注于输入层面，以最小的认知负担实现"说到即所得"。

### 1.2 解决的问题

- macOS 原生语音输入法响应慢、识别差、无法结合 LLM 校正
- 豆包输入法等商业产品需要联网，隐私敏感场景无法使用
- 现有开源方案（如 macOS 原生听写）缺少 LLM 文本校正能力
- 双拼用户需要在语音输入和 Rime 等专业输入法之间切换

### 1.3 核心价值主张

> **"按住说话，松开输入，AI 帮你润色。"**

## 2. 用户画像

### 2.1 主要用户

| 维度 | 描述 |
|------|------|
| **设备** | Apple Silicon Mac（推荐 32GB+ 内存） |
| **模型分发** | 不打包在安装包内，首次运行引导下载 |
| **操作系统** | macOS 13 Ventura 及以上 |
| **输入方式** | 拼音/双拼输入法（Rime 用户为主） |
| **场景** | 程序员、技术写作者、深度 macOS 用户 |
| **痛点** | 不满意苹果原生语音输入，追求效率和隐私 |

### 2.2 用户故事

**作为** 一名程序员
**我希望** 在写代码时能快速用语音输入注释和文档
**这样** 就不用打断键盘输入的节奏，同时语音输入的内容经过 LLM 校正后术语准确

**作为** 一名技术写作者
**我希望** 按住一个键就能语音输入，松开后文字出现在光标处
**这样** 我的思路不会因为切换工具而中断，且输出内容口语化部分被自动整理

## 3. 功能需求

### 3.1 核心功能（Phase 1）

#### F1: 全局 Push-to-Talk

| 属性 | 描述 |
|------|------|
| 触发方式 | 按住全局快捷键（默认：右 Option） |
| 开始 | 按下快捷键 → 开始录音 + 显示 Toast |
| 结束 | 松开快捷键 → 停止录音 → ASR → LLM → 注入文字 |
| 反馈 | Toast 实时显示 ASR 识别文字 |

#### F2: 本地 ASR（Qwen3-ASR-0.6B）

| 属性 | 描述 |
|------|------|
| 模型 | Qwen3-ASR-0.6B，MLX 框架 |
| 推理位置 | 本地（Apple Silicon GPU） |
| 音频格式 | 16kHz, 16-bit, Mono PCM |
| 显存占用 | ~500MB（4-bit 量化） |
| 实时因子 | < 0.01（100 秒音频 < 1 秒处理） |

#### F3: 本地 LLM 校正（Qwen3-4B-4bit）

| 属性 | 描述 |
|------|------|
| 模型 | Qwen3-4B-4bit，MLX 框架 |
| 推理位置 | 本地（Apple Silicon GPU） |
| 显存占用 | ~2.5GB（4-bit 量化） |
| 校正能力 | 删除口水词、同音字纠错、术语修正、通顺化 |

#### F4: 文字注入

| 属性 | 描述 |
|------|------|
| 注入方式 | 剪贴板 + Cmd+V 模拟 |
| 中文兼容 | 自动退出输入法 → 注入 → 恢复状态 |
| 焦点检测 | AXUIElement 获取当前输入框 |

#### F5: 系统托盘

| 属性 | 描述 |
|------|------|
| 图标 | 麦克风图标，录音时变红 |
| 菜单项 | 关于 TNT、设置、退出 |
| 常驻 | 始终在菜单栏运行 |

#### F6: 快捷键配置

| 属性 | 描述 |
|------|------|
| 默认键 | 右 Option 键 |
| 可配置 | 用户可自定义为其他快捷键 |
| 存储 | UserDefaults 持久化 |

#### F7: 设置界面

| 选项 | 默认值 | 可选值 |
|------|-------|-------|
| 全局快捷键 | 右 Option | 用户自定义 |
| ASR 模型 | Qwen3-ASR-0.6B | Qwen3-ASR-0.6B / Apple Speech |
| LLM 校正 | 开启 | 开启 / 关闭 |
| LLM 模型 | Qwen3-4B-4bit | Qwen3-4B-4bit / GLM-4-9B-4bit |
| 模型下载 | — | 下载 / 重试 / 删除本地副本 |

> **注意**：ASR 和 LLM 模型不随安装包分发，用户首次使用前需在设置页面下载。下载链接存储在配置中，支持 HuggingFace 和 ModelScope（国内加速）双源。

### 3.2 Phase 2 功能（规划中）

- **F8: 截图 OCR 上下文**：激活时截取屏幕 → Vision OCR → 作为 LLM Prompt 上下文
- **F9: 剪贴板历史**：监听 NSPasteboard → 最近 50 条 → 作为 LLM Prompt 上下文
- **F10: 云端 API 兜底**：本地模型置信度低时自动调用云端 ASR/LLM

## 4. 非功能需求

### 4.1 性能

| 指标 | 要求 |
|------|------|
| **模型预加载** | App 启动时异步加载，< 30s 完成，不阻塞 UI |
| 热键响应 | 按下 → 录音开始 < 100ms（模型已在内存中） |
| ASR 延迟 | 松手 → ASR 结果 < 1s |
| LLM 延迟 | ASR 结果 → 校正完成 < 2s |
| 文字注入 | LLM 完成 → 文字出现 < 500ms |
| 内存占用 | 总计 < 10GB 统一内存（32GB Mac） |

### 4.2 隐私

- 所有模型推理在本地完成，不上传任何音频或文字到网络
- 截图 OCR 使用 Vision 框架，完全离线
- 剪贴板历史仅存储在本地内存，App 退出时清除

### 4.3 兼容性

- macOS 13.0+（Ventura）
- Apple Silicon（推荐）/ Intel（性能较差）
- Rime（鼠须管）并行使用无冲突

## 5. 交互设计

### 5.1 核心交互流

```
用户按下右 Option
    │
    ├─ Toast 出现（"倾听中..."，渐入动画 300ms）
    ├─ 托盘图标变红
    └─ 麦克风图标开始波动

用户说话中
    │
    ├─ Toast 显示实时 ASR 识别结果（刷新间隔 500ms）
    └─ 麦克风图标持续波动

用户松开右 Option
    │
    ├─ Toast 显示 "校正中..."
    ├─ 托盘图标闪烁
    ├─ ASR 全量结果 → LLM 校正
    └─ Toast 显示校正后结果（短暂停留 1s）

文字注入
    │
    ├─ Toast 消失（淡出 300ms）
    ├─ 文字出现在输入框
    └─ 托盘图标恢复白色
```

### 5.2 Toast 设计

| 属性 | 值 |
|------|---|
| 尺寸 | 宽度 400px，高度自适应（最大 80px） |
| 背景 | 黑色半透明（`rgba(0,0,0,0.75)`） |
| 圆角 | 12px |
| 位置 | 当前焦点窗口下方 20px，水平居中 |
| 字体 | SF Pro Text，14pt，白色 |
| 状态文字 | "倾听中..."（橙色）、"校正中..."（蓝色）、错误（红色） |

### 5.3 菜单栏图标

| 状态 | 图标 |
|------|------|
| 空闲 | 白色麦克风 |
| 录音中 | 红色麦克风 + 脉冲动画 |
| 异常 | 灰色麦克风 + 感叹号 |

### 5.4 设置界面

采用 macOS 标准设置界面布局（TabView）：
- **通用**：快捷键配置、关于
- **模型**：ASR/LLM 模型下载管理、下载进度、存储路径
- **高级**：模型路径、上下文开关

## 6. 技术架构

### 6.1 分层架构

```
┌─────────────────────────────────┐
│        用户交互层                 │
│   菜单栏 · Toast · 设置界面       │
├─────────────────────────────────┤
│        核心控制层                 │
│   状态机 · 热键管理 · 权限管理     │
├─────────────────────────────────┤
│        AI 引擎层                  │
│   ASR(Qwen3-ASR) · LLM(Qwen3-4B) │
│   音频处理 · OCR(Vision)          │
├─────────────────────────────────┤
│        系统集成层                 │
│   CGEventTap · 输入焦点 · 剪贴板   │
└─────────────────────────────────┘
```

### 6.2 数据流

```
快捷键按下
  → AudioEngine.startRecording()
  → PCM Buffer 循环送入 ASREngine
  → ASREngine.streamTranscribe() → interim text → Toast.updateText()
  ↓
快捷键松开
  → AudioEngine.stopRecording()
  → ASREngine.finalTranscribe() → raw text
  → LLMRefiner.refine(raw_text, context) → corrected text
  → FocusManager.inject(text)
  → Toast.hide()
```

### 6.3 关键模块

| 模块 | 职责 | 关键 API |
|------|------|---------|
| `HotkeyManager` | CGEventTap 热键监听 | `start()`, `stop()`, `onKeyDown`, `onKeyUp` |
| `AudioRecorder` | AVAudioEngine 音频采集 | `start()`, `stop()`, `onBufferReady` |
| `ASREngine` | Qwen3-ASR MLX 推理 | `streamTranscribe()`, `finalTranscribe()` |
| `LLMRefiner` | Qwen3-4B MLX 推理 | `refine(text, context)` |
| `FocusManager` | 焦点检测 + 文字注入 | `getFocusedElement()`, `inject(text)` |
| `ToastWindow` | 悬浮提示 UI | `show(text, state)`, `update(text)`, `hide()` |
| `SettingsStore` | 配置持久化 | `load()`, `save()`, `onChanged` |

## 8. 成功指标

### 8.1 定量指标

| 指标 | 目标 |
|------|------|
| 端到端延迟（中文字） | < 3s |
| ASR 识别准确率（安静环境） | > 90% |
| LLM 校正满意度（主观评分） | > 4/5 |
| App 内存占用（空闲） | < 200MB |
| App 内存占用（录音中） | < 2GB |

### 8.2 定性指标

- 用户不觉得"切换语音输入"是一件麻烦事
- 用户愿意在日常工作中使用 TNT 替代部分键盘输入
- 双拼用户可以完全不退出 Rime 就完成语音输入

## 9. 风险与缓解

| 风险 | 概率 | 影响 | 缓解方案 |
|------|------|------|---------|
| CGEventTap 权限被拒 | 中 | 高 | 提供清晰的权限引导弹窗 |
| MLX Swift API breaking change | 低 | 中 | 锁定 mlx 版本，隔离封装层 |
| ASR 识别质量差 | 中 | 中 | LLM 校正兜底 + 云端 API 可选 |
| 模型下载失败（国内网络） | 中 | 中 | 提供 modelscope.cn 备选源 |
| 内存不足（16GB Mac） | 低 | 中 | 限制同时加载的模型数量 |
